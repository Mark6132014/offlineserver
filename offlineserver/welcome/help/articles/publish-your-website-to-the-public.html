Publish Your Website To The Public

In OfflineServer

Publishing your website to the public with OfflineServer is quite simple.
If you want your website to be found locally only in your network, your private IP address is your default website address people can access.
OfflineServer is simple, it gives you a free DNS (Domain Name Server) hostname that your local DNS Server can add.
If you want your website to the public, you need a public IP attached to the computer that is hosting the OfflineServer Website.

Your port forwaring should be like this:

SERVICE NAME: WHATEVER YOU WANT TO NAME IT


EXTERNAL PORT: 80 (or) 443


INTERNAL PORT: 80 (or) 443


INTERNAL IP: (the ip address that is hosting the OfflineServer Website)


SOURCE IP: (your public ip or leave it blank for better compatibility)


That's how you do it. You can use a Windows Server as a DNS Server to assign a PUBLIC hostname.


Your DNS Server should have this in "Forward Lookup Zones"

Forward lookup zone name: (YOUR WEBSITE HOSTNAME LIKE offlineexample.org)


Should have a A record in the forward lookup zone, no dynamic updates for the forward lookup zone please, and make the forward lookup zone primary


A Record Name: nothing (if you want it as a subdomain you can)


A Record FQDN should be the same as your forward lookup zone name (or your public hostname if you want to call it that way)


A Record IP address: (the ip address that is hosting your offlineserver website, if the ip is tied to a public ip too, use that for good public use)


You are done!


If you assign a free domain to your OfflineServer website, it assigns it in a system file called "hosts". It uses 127.0.0.1 to point to the real ip address you want


to host your offlineserver website at. This will really work in Windows Mobile Hotspot. If you have a free hostname (like offlineexample.org) assigned and you turn on


Mobile Hotspot to the host of your offlineserver website, offlineexample.org would work for anyone who connects to your mobile hotspot!
